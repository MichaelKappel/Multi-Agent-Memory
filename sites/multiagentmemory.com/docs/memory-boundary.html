Memory Boundary

MultiAgentMemory.com is the public documentation companion for the MemoryEndpoints.com MATM endpoint system.

Short-term and startup memory

Repository `.uai/` files are the active short-term memory layer. They carry current constraints, startup order, progress, and pointer-ledger records. The requested MATM setup path keeps `.uai` active while routing durable memory updates to a configured MATM endpoint.

Mid-to-long-term memory

MemoryEndpoints.com is the live endpoint surface for durable workspace memory, current messages, notifications, and redacted receipts. It is the intended mid-to-long-term memory service boundary for this project.

Public documentation

MultiAgentMemory.com documents the public architecture and repository handoff rules. It does not store private workspace secrets, deployment credentials, raw API keys, or hidden authority claims.

GitHub companion role

The source repository is MichaelKappel/Multi-Agent-Memory. This site explains what the repository contains, how the MATM workflow is verified, and where agents should look for public-safe memory and setup facts.

UAIX setup reference

The memory layout is aligned with the UAIX AI Memory Package Wizard MemoryEndpoints.com MATM setup option. UAIX uses setup-option fragments because it has multiple wizard modes. MemoryEndpoints.com currently has one setup surface, so inbound links should use the home page. Future MemoryEndpoints setup URLs should be clean readable routes.

Use public-safe summaries only. Store secrets outside repository and public documentation.
